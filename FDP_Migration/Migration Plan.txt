The FDP stores its (internal) application data in mongodb.

This includes references to its domain, in URIs in the ACL and metadata collections.

So, these need to be updated too.

Unfortunately, there is no built-in solution for this yet, so we need to update the mongodb content manually.

Here's a quick and dirty way to do this using the mongodb shell (mongosh):

- log in to your host
- tear down the fdp container, if it is running
    docker compose down fdp
- open a mongosh shell in the running mongodb container:
    docker compose exec mongo mongosh
- activate the FDP database, typically called fdp (you can list all databases using e.g. show dbs):
    use fdp
- [optional] inspect the ACL and metadata collections to verify that the old domain is present:
     db.ACL.find()
    db.metadata.find()
- update the URIs in the ACL collection:
    db.ACL.updateMany(
      {instanceId: {$regex: /<old-uri-pattern>/}},
      [{$set: {instanceId: {$replaceOne: {input: "$instanceId", find: "<old-uri>", replacement: "<new-uri>"}}}}]
    )
- update the URIs in the metadata collection:
    db.metadata.updateMany(
       {uri: {$regex: /<old-uri-pattern>/}},
       [{$set: {uri: {$replaceOne: {input: "$uri", find: "<old-uri>", replacement: "<new-uri>"}}}}]
    )
- inspect the collections again to verify the changes (see 5.)
- exit the mongodb shell:
    .exit
- set up the fdp container again:
     docker compose up -d fdp
